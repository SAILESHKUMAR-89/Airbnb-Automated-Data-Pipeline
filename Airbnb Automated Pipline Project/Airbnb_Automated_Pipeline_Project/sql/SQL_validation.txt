### SQL Validation Summary

The processed Airbnb dataset was loaded into Oracle using **SQL*Loader (.ctl)** for validation of pipeline outputs.
During the loading process, **246,412 rows were successfully inserted**, while **1,866 rows were rejected due to data errors**.
The primary reason for rejected rows was the **`amenities` column**, which contains long text strings with special characters and Unicode symbols. 
These characters caused parsing issues during the SQL*Loader process because Oracle expects strict CSV formatting and 
may fail when encountering complex text structures or encoding inconsistencies.
Since the rejected rows represent **less than 1% of the total dataset**, the loaded data was still considered **valid and sufficient for SQL validation checks**.
After loading, key metrics such as **row counts, average price, host distribution, and listing categories** were verified using SQL queries 
to ensure the pipeline transformations were correctly applied.
Therefore, despite minor row rejections caused by text encoding in the `amenities` field, the dataset successfully passed **SQL validation for analytical consistency.**
